# 软件需求规格说明书 (SRS) - 智能审查助手

## 1. 引言
### 1.1 项目背景
在企业内控和合规管理中，业务方案、合同及制度的审查是一项高频且严谨的工作。传统人工审查存在效率低、规则覆盖不全、标准不统一等痛点。本系统旨在利用大语言模型（LLM）的深度理解能力，结合企业本地规则库，构建一个智能化的辅助审查工具。

### 1.2 文档目的
本文档详细描述了智能审查助手的各项功能及非功能需求，为后续开发、测试及验收提供基准。

## 2. 角色定义
| 角色 | 描述 | 主要职责 |
| :--- | :--- | :--- |
| **审查专家 / 管理员** | 业务规则的制定者 | 维护规则知识库、上传标准制度、启停特定规则。 |
| **业务人员 / 审查员** | 实际业务的执行者 | 上传待审材料、运行 AI 智能分析、获取审查意见。 |

## 3. 功能需求

### 3.1 规则知识库 (Rule Base)
*   **FR-1.1 规则录入**：支持手动录入规则标题、正文及分类（法律、财务、合规、技术）。
*   **FR-1.2 文档导入规则**：支持通过上传 .pdf, .docx, .txt 文件自动提取文本并填充为新规则。
*   **FR-1.3 规则调度管理**：用户可一键开启或关闭特定规则，仅“启用”状态的规则会进入 AI 的上下文窗口。
*   **FR-1.4 规则预览**：提供沉浸式模态框，支持对长篇幅规则的完整阅读。

### 3.2 审查工作台 (Review Workspace)
*   **FR-2.1 多格式材料上传**：支持对待审查的业务方案进行多文件上传（支持 PDF、DOCX、TXT）。
*   **FR-2.2 文档解析策略**：针对 PDF 提供多模态（Base64）和纯文本（OCR/Text Extraction）双重解析，以适配不同模型需求。
*   **FR-2.3 智能摘要生成**：
    *   **FR-2.3.1 自动摘要**：提取方案核心目标、范围及主要干系人。
    *   **FR-2.3.2 提示词引导**：支持用户输入“关注点提示词”，AI 需根据指令调整摘要权重。
*   **FR-2.4 风险点提取 (Extraction)**：依据规则库，结构化提取文档中的关键数据点，并判定风险等级（高/中/低）。
*   **FR-2.5 审查意见草拟**：基于风险提取结果，自动生成包含“审查结论、合规分析、风险警示、整改建议”的专业文书。

### 3.3 系统底层支撑
*   **FR-3.1 双模型引擎适配**：支持 Google Gemini 3.0 系列与 DeepSeek V3 引擎的实时无缝切换。
*   **FR-3.2 国际化 (i18n)**：全界面支持中英文双语切换，包括 AI 输出的语言指令。

## 4. 非功能需求
*   **NFR-4.1 响应速度**：摘要及提取操作应在 15 秒内给出初步反馈（受大模型 API 延迟影响除外）。
*   **NFR-4.2 数据隐私**：所有 API Key 均存储在环境变量中，前端直接请求模型端点，不经过任何第三方中间服务器存储。
*   **NFR-4.3 健壮性**：对于扫描件 PDF 等无法提取文本的情况，系统应给出明确提示，不影响其他文档的处理。
